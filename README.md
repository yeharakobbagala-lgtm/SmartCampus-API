📌 Project Overview
SmartCampus is a professional-grade RESTful API developed using Java EE (JAX-RS). The system is designed to manage university infrastructure by tracking Rooms, Sensors (CO2, Temperature), and real-time Readings.

The project demonstrates advanced backend concepts including HATEOAS, sub-resource delegation, global exception mapping, and observability through logging filters.

🚀 Key Features by Phase
Phase 1: Discovery & Hypermedia
HATEOAS Implementation: A Discovery Endpoint provides entry points to the API via dynamic JSON links.

Standardized Responses: Uses consistent JSON structures for API metadata.

Phase 2: Room Management
CRUD Operations: Full support for Creating, Reading, Updating, and Deleting rooms.

In-Memory Storage: Uses a thread-safe Mock Database with a Singleton DAO pattern.

Idempotency: Implements idempotent DELETE operations for reliability.

Phase 3: Sensor Operations & Search
Linking: Sensors are logically linked to Rooms via ID references.

Search & Filtering: Supports searching sensors by type (e.g., ?type=CO2) using JAX-RS Query Parameters.

Integrity Checks: Validates that a Room exists before allowing a Sensor to be added (422 error handling).

Phase 4: Sub-Resources (Readings)
Deep Nesting: Implements the Sub-Resource Locator pattern for paths like /sensors/{id}/readings.

State Management: Adding a new reading automatically updates the lastReading value on the parent Sensor.

Phase 5: Advanced Error Handling & Logging
Leak-Proof API: Custom Exception Mappers prevent Java stack traces from reaching the user.

Observability: A Global Logging Filter records every incoming request (Method/URI) and outgoing response (Status Code) in the server logs.

Semantic Errors: Returns accurate status codes like 409 Conflict (for busy rooms) and 422 Unprocessable Entity.

🛠️ Technical Stack
Language: Java 8+

Framework: JAX-RS (Jersey Implementation)

Server: GlassFish Open Source Edition 5.1.0

Format: JSON (Jackson Provider)

Build Tool: Apache Ant / NetBeans IDE

📥 Installation & Running
Clone the project into your NetBeans workspace.

Right-click the project and select Clean and Build.

Run the project (Ensure GlassFish server is started).

The API will be available at:

http://localhost:8080/SmartCampus/api/v1/

🧪 Testing with Postman
A full testing suite is included. Recommended test sequence:

GET /v1/ - View Discovery Links.

POST /v1/rooms - Create a room (e.g., LIB-301).

POST /v1/sensors - Add a sensor to the created room.

POST /v1/sensors/1/readings - Submit sensor data.

DELETE /v1/rooms/LIB-301 - Test 409 Conflict (cannot delete room with sensors).

🛡️ Security & Observability
Error Masking: All internal server errors (500) are intercepted to hide system architecture from attackers.

Audit Logs: Check the GlassFish Server Output to see the real-time request/response logs generated by the ApiLogger.
